# Project title

## 1. Question
State the real problem in one or two sentences.

## 2. Background research
Define the concepts you had to research. Add the sources used in the project README.

## 3. Assumptions
List every important assumption, unit, convention, and simplification.

## 4. Inputs
Describe the required user inputs or dataset columns.

## 5. Mathematics
Write the formulas before implementing them. Define every symbol.

## 6. Program design
Plan the functions, files, and data structures.

Suggested structure:

```text
project_name/
├── README.md
├── main.py
├── model.py
├── data/
├── outputs/
└── tests/
```

Use only the files that the project genuinely needs.

## 7. Core requirements
- [ ] input or data validation;
- [ ] reusable functions;
- [ ] correct calculations;
- [ ] meaningful output;
- [ ] tests for the main logic;
- [ ] interpretation of results.

## 8. Edge cases
List cases that could break the model, such as missing data, zero denominators, negative values, invalid weights, or inconsistent units.

## 9. Results
Present the principal numbers, tables, and charts.

## 10. Interpretation
Explain what the results mean in finance, mathematics, economics, or energy terms.

## 11. Limitations
Explain what the model ignores and why its conclusions should not be overstated.

## 12. Extensions
List improvements for a future version. Do not implement every extension in version 1.

## 13. Completion checklist
- [ ] I can explain every formula.
- [ ] I can explain every function.
- [ ] The program handles invalid input.
- [ ] The results have been checked manually on a small example.
- [ ] The README explains assumptions and limitations.
- [ ] The project runs from a clean terminal command.
- [ ] I committed the project in meaningful stages.
